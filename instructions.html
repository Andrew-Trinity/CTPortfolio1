GitHub Desktop — Student Guide

Use this page to publish and update your portfolio using GitHub Desktop.

Install GitHub Desktop

  1. Go to desktop.github.com and download for Windows.
  2. Run the installer and finish setup.
  3. When asked, allow it to install Git — no extra steps needed.
Tip: If you already have it, open it from the Start Menu.

Sign in to GitHub

  1. Open GitHub Desktop.
  2. Click Sign in to GitHub.com and log in in the browser.
  3. Return to Desktop and click Continue.

Create a Repository on GitHub

  1. In a browser, go to github.com → + (top-right) → New repository.
  2. Name it like 10CT-Portfolio. Leave README, .gitignore, and License unchecked.
  3. Click Create repository.

Add Your Local Folder

  1. Download the starter folder (or use your own project) and keep everything together.
  2. In GitHub Desktop: File → Add local repository…
  3. Choose your folder (e.g., Desktop → WebDev → Project1-10CT-Portfolio) and click Add repository.

Publish to GitHub

  1. In GitHub Desktop, click Publish repository.
  2. Choose Public or Private. Keep the name the same.
  3. Click Publish Repository.

Now refresh your repo page on GitHub — your files should be online.

Everyday Workflow

  1. Save your changed files.
  2. Open GitHub Desktop: review changed files on the left.
  3. Write a short Summary (e.g., "Add navbar styling").
  4. Click Commit to main.
  5. Click Push origin (top-right) to upload to GitHub.

Common Fixes

  • Don’t see files online? Click Push origin in GitHub Desktop.
  • Wrong folder? File → Remove repository, then Add local repository… again.
  • Publish button greyed out? Check Repository → Repository Settings… for the GitHub link.
